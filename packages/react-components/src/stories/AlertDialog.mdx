## Slots

Alert Dialog has 3 content slots, populated via prop:

- `title`: the bold heading at the top
- `description`: the body text in the middle
- `buttons`: renders a [ButtonGroup](?path=/docs/components-button-buttongroup--docs) at the bottom, and expects an array of one or more [Buttons](?path=/docs/components-button-button--docs)

If no content is passed to a slot, it will not be rendered.

## Variants

Alert Dialog ships with a set of themes, set via the `variant` prop (defaults to `info`). Variants render different icons and colour schemes for common use cases.

## Additional customisation

Set `isCloseable` to `false` to hide the close button in the top right corner of an alert. Use this in conjunction with [Modal's](?path=/docs/components-dialogs--docs#modal) `isDismissable` and `isKeyboardDismissDisabled` props to control whether the user can close a dialog without taking action:

Set `isIconHidden` to `true` to hide the variant icon to the left of the dialog title:

<Canvas of={AlertDialogStories.AlertDialogWithoutIcon} />

### ARIA roles

By default, an Alert Dialog renders with the [ARIA "dialog" role](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/dialog_role).

If needed, you can set [the more assertive alertdialog role](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/alertdialog_role) using the `role` prop.
