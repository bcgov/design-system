# Subheader

<Subtitle>
  A subheader displays a list of links, menus or buttons to help the user
  navigate an interface.
</Subtitle>

## Usage and resources

<Source
  code={`import { Subheader } from "@bcgov/design-system-react-components";`}
  language="typescript"
/>

Subheader is a simple container component that renders a list of [links](/docs/utility-text-and-links-link--docs), [menus](/docs/navigation-and-structure-menu--docs) and/or [buttons](/docs/inputs-and-controls-button--docs). It is primarily intended to be used in parallel with the [Header](/docs/navigation-and-structure-header--docs) component.

Learn more about working with the Subheader component:

- [Usage and best practice guidance](https://www2.gov.bc.ca/gov/content?id=AE80414124294EADA75D7853A96BD2E5)
- [View the subheader component in Figma](https://www2.gov.bc.ca/gov/content?id=8E36BE1D10E04A17B0CD4D913FA7AC43#designers)

## Configuration

Subheader renders as a `<div>` containing a `<nav>` element. It automatically renders a [Separator](/docs/utility-separator--docs) between each child item.

The primary use case for a Subheader is alongside the Header component:

<Canvas of={SubheaderStories.HeaderWithSubheader} />

**Note**: Subheader has the ARIA ["navigation"](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Reference/Roles/navigation_role) role. When using it alongside other navigation landmarks (like the Header), you should also set a descriptive name using the `ariaLabel` prop.

### Sizing

Set `size` to `small` to reduce the amount of vertical padding and spacing between individual items.
